# Stockholm Golf Price Dashboard

Static dashboard for tracking the cheapest known green fee or tee-time price over the coming 60 days for the Stockholm golf courses from the requested That's up guide.

Open the dashboard through a local web server so the page can read `data/courses.json` and `data/prices.csv`:

```powershell
npm run serve
```

## Price Data

`data/prices.csv` is generated by the Min Golf fetcher and deployed by the scheduled GitHub Actions workflow.

Required columns:

```csv
course_id,course_name,date,price,currency,tee_time,holes,source_url,note
```

The dashboard ignores rows outside the next 60 days and computes the cheapest known row per course.

When a Min Golf club has more than one course, the fetcher keeps only 18-hole courses where that can be determined. The selected Min Golf course name is written to `course_name` and displayed under the club name.

## Live Prices

Many Stockholm clubs route booking and dynamic prices through Min Golf or club booking portals that may require login. The included fetcher uses your configured Min Golf credentials locally or in GitHub Actions and writes normalized rows into `data/prices.csv`.

## Fetch From Min Golf

Create `.env.local` from `.env.local.example`:

```env
MINGOLF_USERNAME=your-golf-id-or-email
MINGOLF_PASSWORD=your-password
```

Then run:

```powershell
.\scripts\run-mingolf-fetch.ps1
```

The fetcher logs in locally with Playwright, looks for booking/green-fee responses, and writes recognized rows to `data/prices.csv`. If Min Golf's current UI/API needs tuning, it writes a safe discovery report to `data/mingolf-discovery.json` without printing your credentials.

## Scheduled Deploy

This repo includes a GitHub Actions workflow at `.github/workflows/scrape-and-deploy.yml`.

It runs automatically at 21:00 Europe/Stockholm time every day, scrapes Min Golf, builds a static site artifact, and deploys it to GitHub Pages.

GitHub setup:

1. Push this folder to a GitHub repository.
2. In GitHub, go to `Settings` -> `Secrets and variables` -> `Actions`.
3. Add repository secrets:
   - `MINGOLF_USERNAME`
   - `MINGOLF_PASSWORD`
4. Go to `Settings` -> `Pages`.
5. Set `Build and deployment` source to `GitHub Actions`.
6. Run `Scrape and deploy` manually once from the `Actions` tab, or wait for the next 21:00 scheduled run.

The workflow deploys only `index.html`, `app.js`, `styles.css`, `assets/`, `data/courses.json`, and `data/prices.csv`. It does not deploy `.env.local`, temporary files, or discovery logs.
